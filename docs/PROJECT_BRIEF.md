# Project Brief

## Working name

**Chamber Clerk Bot**

## Purpose

Chamber Clerk Bot is a Discord bot MVP for a digital-first **International Private Law Chamber**.

The bot exists to help a Discord server operate like a private-law chamber: members can apply, join committees, submit motions, draft clauses, adopt rules, archive doctrine, manage arbitration case metadata, and run a Special Arbitration Review process.

## Digital-first strategy

The project starts as a Discord server instead of a physical institution.

The long-term vision may include a formal institution, arbitration chamber, review chamber, offices, boardrooms, committee rooms, law library, archive, secure meeting spaces, and physical headquarters. Those are future-stage possibilities. The MVP is digital-first.

## Core idea

Public law governs citizens through state authority. Private law governs consenting parties through contract, membership, association, entity rules, arbitration agreements, and voluntary legal instruments.

The Chamber specializes in the second category.

## What the bot should prove

The MVP should prove that a private-law chamber can be coordinated digitally through:

- structured membership
- committee governance
- motion submission
- private-law drafting
- adopted rule archives
- arbitration registry metadata
- review panel workflows
- markdown exports
- audit logs

## What the bot must not do

The bot must not create or imply:

- sovereignty
- public-law authority
- criminal law
- policing power
- taxation power
- public court authority
- election authority
- compulsory jurisdiction over non-consenting people

## First implementation goal

Create one modular Discord bot using Python and discord.py with slash commands only.

The bot should be simple, stable, and extendable rather than overbuilt.
